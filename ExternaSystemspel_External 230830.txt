1 INTRODUCTION  1.1 Purpose “Externa Systemspel” is a service on Svenska Spel's site where customers can place wagers using text files in a special format or by calling some of the APIs that Svenska Spel offers.  A general introduction and some detailed information regarding the service can be found on respective web sites (login required). That’s also where upload of text files is done. Navigate to https://spela.svenskaspel.se click on “Mer” and select “Externa systemspel” 1.2 Scope This document describes the APIs that are available for placing wagers towards Svenska Spel. The information is mainly addressed to system developers and similar roles who master and understand the underlying principles and terms used in this document.  I.e. the document will not cover the principles behind how clients on the Internet download and send data, or what https means. 2 API Versions 2.1 Version numbers Svenska Spels external API:s use a numeric versioning system in its URL:s. If breaking changes must be implemented in an API, that API will receive a new URL with an increased version number leaving the previous API version unchanged. 2.1.1 Example Let’s say the API to fetch current draws for product Matchen needs a change that breaks the API contract. To get the new features, use the new URL with an increased version number like: https://api.www.svenskaspel.se/external/2/draw/matchen/draws (notice the 2 in the URL) At the same time the previous version still functions just as before on the unchanged URL: https://api.www.svenskaspel.se/external/1/draw/matchen/draws

2.2 API removal If there are any reason where it is not possible to maintain an old version, we will give notice in advance for you to update your software well before an old API version is removed. 2.3 Content Type Content-Type and charset must always enter in the post header: application/json;charset=utf-8 For example: https://api.www.svenskaspel.se/external/1/wager/ownwager/stryktipset?accesskey=test7b9de3-5cbf-37c8-8n34-test3df954f8f Content-Type: application/json;charset=utf-8   2.4 API-key The api-key only needs to be sent one time for each document.  3 Use case 3.1 Upload When the customer chooses to upload the wager, the web client (Svenska Spel's web service, the service where Svenska Spel's customers can play on the products that Svenska Spel offers) creates a JSON document and sends it as an http-post to the generic url: https://api.www.svenskaspel.se/external/1/wager/ownwager/{product_name}?accesskey=??? Content-Type: application/json;charset=utf-8  3.2 Verify When the upload of the wager is complete, a reply is sent back (see below) with a URL that the customer must use to acknowledge their wagers.  The customer must go to this URL within 30 minutes, otherwise the uploaded wagers will be deleted. Please note that this URL may change so it is important that it is not hardcoded by the software manufacturer. The customer will get a login prompt and can after that chose to pay or cancel the wagers.

3.3 Personal or Group wagers When paying for an uploaded bet the customers can either place the bet towards their own account and available balance or towards a balance of a specific group where the customer is a member (with relevant permissions). The service for placing bets on behalf of a group is named “Spela Tillsammans” at Svenska Spel.  Please note that different size limits on the bets is applied due to technical reasons.  3.4 Limits  There is a limit of how many items a wager can contain and a max wager amount of 20 000 SEK. If any of these limits exceed the customer get an error message and the upload fails. 3.4.1 Personal wager 3.4.1.1 Stryktipset, Europatipset Single item:  10.000 M-system: 1.000 3.4.1.2 Topptipset, Powerplay  Single item:  1.000 M-system: 100 3.4.1.3 Måltipset Single item:  10.000 (8.000 reach the limit of 20.000 SEK)  3.4.1.4 Bomben Single item:  10.000 3.4.1.5 Fullträff Single item:  10.000 M-system:  10.000  3.4.2 Group wagers (Spela Tillsammans)
